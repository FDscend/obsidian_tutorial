# Obsidian 教程项目总规划

## 1. 项目目标

本仓库用于产出一套可协作维护的教程，满足以下目标：

1. 新手可从零完成环境安装与协作配置（Obsidian + VS Code + Git/GitHub）。
2. 学员可系统掌握 Markdown 基础、扩展语法与 Obsidian 特性（含 Canvas、Bases）。
3. 团队可使用 VS Code 的 Git 可视化流程进行协作提交与冲突处理。
4. 学员可完成 Syncthing 局域网同步仓库的部署。
5. 学员可掌握 VS Code Copilot 的基础使用与进阶案例（prompt、skill、mcp）。
6. 课程结构清晰，支持后续由 agent 分章节持续迭代。

## 4. Section 级教学设计

### section0：课程导读与协作规范

- 学习目标：理解课程全景、协作约定、提交流程。
- 输入资料：README、仓库目录、团队协作约定。
- 产出物：协作规范文档、学习路径图。
- 验收标准：新成员可在 10 分钟内明确学习顺序与提交流程。

### section1：环境安装（Obsidian / VS Code / Git / GitHub）

- 学习目标：完成工具安装、账号配置、插件准备，确保扩展语法和样式渲染正常。
- 输入资料：`环境配置.md`。
- 产出物：分工具安装文档 + 配置检查清单。
- 验收标准：新机器可在 30-45 分钟内完成环境上线，并正确预览仓库文档。

### section2：Markdown 基础

- 学习目标：掌握标题、段落、列表、链接、图片、代码、引用等基础语法。
- 输入资料：`markdown格式总结.md` 的基础语法部分。
- 产出物：基础语法讲解 + 练习题。
- 验收标准：能独立完成一篇规范化笔记。

### section3：Markdown 扩展与 Obsidian 语法

- 学习目标：掌握表格、脚注、任务列表、Mermaid、数学公式、Callout、WikiLink。
- 输入资料：`markdown格式总结.md` 的扩展和 Obsidian 语法部分。
- 产出物：扩展语法章节 + 小练习。
- 验收标准：可产出含图表、公式、脚注的完整笔记。

### section4：Git/GitHub 协作（VS Code 可视化）

- 学习目标：学会克隆、暂存、提交、同步、分支切换、冲突处理。
- 输入资料：`环境配置.md` 的 Git 部分。
- 产出物：图形化操作流程文档 + 冲突演练案例。
- 验收标准：完成一次独立分支开发到合并。

### section5：Obsidian Canvas

- 学习目标：掌握 Canvas 画布创建、卡片连接、内容组织。
- 输入资料：Obsidian 官方文档（zh/help）。
- 产出物：Canvas 入门教程 + 课程地图案例。
- 验收标准：可用 Canvas 搭建一张“知识结构图”。

### section6：Obsidian Bases

- 学习目标：掌握 Bases 数据组织、视图、筛选、排序的基础操作。
- 输入资料：Obsidian 官方文档（zh/help）。
- 产出物：Bases 入门教程 + 教程内容数据库案例。
- 验收标准：可建立一个教程条目数据库并进行筛选。

### section7：Syncthing 局域网同步

- 学习目标：完成 Syncthing 双端安装、设备配对、文件夹共享与排障。
- 输入资料：官方文档 + 本仓库协作约定。
- 产出物：部署教程、同步策略、故障排查清单。
- 验收标准：两台设备在局域网内成功同步仓库。

### section8：VS Code Copilot 基础

- 学习目标：掌握官方订阅、扩展安装、基础对话与代码协作。
- 输入资料：Copilot 官方文档 + 本仓库指令约定。
- 产出物：基础使用手册（含 instruction/prompt/skill/mcp 概览）。
- 验收标准：学员可独立完成一次“提需求 -> 生成草稿 -> 修改落地”的流程。

### section9：Copilot 实战案例

- 学习目标：掌握 instruction、prompt、skill、mcp 在真实任务中的组合用法。
- 输入资料：案例需求与开源 skill 参考。
- 产出物：案例文档（含 Tavily MCP 搜索、Skill 制作与安装）。
- 验收标准：可复现实战案例并迁移到自定义任务。

## 5. 编写规范（适配 Obsidian + VS Code）

1. 所有教程正文使用 UTF-8 编码的 Markdown。
2. 每个章节首部使用统一 Frontmatter：title、tags、status、updated。
3. 每节包含固定段落：学习目标、前置条件、步骤、常见问题、练习任务、验收清单。
4. 图片统一放在 `image/` 或章节内 `assets/` 子目录。
5. 示例命令优先提供 VS Code 可视化路径；命令行作为补充。
6. 需要平台差异时明确标注 Windows/macOS/Linux。

## 6. Agent 推进工作流（后续执行标准）

后续让 agent 按以下节奏推进：

1. 一次只推进一个 section（避免跨节并行导致风格漂移）。
2. 每次推进步骤：
   - 读取本规划文档与该 section 的现有文件。
   - 产出该 section 的章节草稿（至少 1-2 个 md）。
   - 自检：术语一致性、链接可用性、步骤可复现。
   - 更新 `appendix/C_更新日志.md`。
3. 每次提交信息格式建议：
   - `docs(sectionX): 新增/完善 xxx 教程`
4. 验收通过后，再进入下一个 section。
