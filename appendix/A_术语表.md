# 术语表

## Markdown 相关

| 术语                  | 说明                                                                                                                                                                                   |
| :-------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Markdown**          | 轻量级标记语言，由 John Gruber 于 2004 年创建，用于在纯文本文档中添加格式化元素。                                                                                                      |
| **Frontmatter**       | 笔记文件顶部的 YAML 元数据区，用 `---` 包裹，可定义 `tags`、`title`、`status` 等属性。Base 的数据来源于笔记的 Frontmatter。                                                            |
| **Callout**           | Obsidian 核心功能，以 `> [!类型]` 语法创建不同类型的提示框。支持 `note`、`tip`、`warning`、`danger`、`info`、`success`、`question`、`bug`、`example`、`quote` 等类型，以及可折叠变体。 |
| **WikiLink**          | Obsidian 内部链接格式，使用 `[[笔记名]]` 或 `[[笔记名\|显示文字]]` 链接到其他笔记、标题或块。                                                                                          |
| **链接嵌入**          | 在 WikiLink 前加 `!` 嵌入笔记内容、图片、音频、视频或 PDF，如 `![[图片.png\|200]]`。                                                                                                   |
| **Mermaid**           | 基于文本的图表工具，用 ` ```mermaid ` 代码块创建流程图、时序图、甘特图等。                                                                                                             |
| **高亮（Highlight）** | Obsidian 特有语法，用 `==文字==` 实现荧光笔效果。                                                                                                                                      |
| **脚注（Footnote）**  | 用 `[^1]` 在正文标记，文末 `[^1]: 内容` 定义，添加注释而不干扰正文阅读。                                                                                                               |
| **任务列表**          | `- [ ]` 未完成 / `- [x]` 已完成。Obsidian 扩展支持 `[-]`（已取消）、`[/]`（进行中）等状态。                                                                                            |
| **围栏代码块**        | 用三个反引号包裹代码块，可指定语言启用语法高亮。                                                                                                                                       |
| **数学公式（LaTeX）** | 用 LaTeX 语法编写数学公式，行内 `$E=mc^2$`，块级 `$$...$$`。VS Code 用 KaTeX，Obsidian 用 MathJax 渲染。                                                                               |
| **转义字符**          | 在 Markdown 特殊字符前加反斜杠 `\` 显示其字面含义。                                                                                                                                    |

## Git / GitHub 相关

| 术语                              | 说明                                                                                                        |
| :-------------------------------- | :---------------------------------------------------------------------------------------------------------- |
| **工作区（Working Directory）**   | 当前正在编辑的文件，尚未暂存。                                                                              |
| **暂存区（Stage / Index）**       | 准备纳入下一次提交的文件列表，允许选择性提交。                                                              |
| **本地仓库（Local Repository）**  | 已提交到本地的 Git 历史记录，存储在 `.git` 目录中。                                                         |
| **远程仓库（Remote Repository）** | GitHub 等远端服务器上的仓库副本。                                                                           |
| **Commit（提交）**                | 将暂存区的更改保存到本地仓库，形成一次历史记录。                                                            |
| **Push（推送）**                  | 将本地仓库的提交推送到远程仓库。                                                                            |
| **Pull（拉取）**                  | 从远程仓库拉取最新更改并合并到本地。                                                                        |
| **Fetch（取回）**                 | 从远程仓库下载数据但暂不合并。                                                                              |
| **Clone（克隆）**                 | 将远程仓库完整复制到本地。                                                                                  |
| **Fork**                          | 在 GitHub 上复制他人仓库到自己的账号下，可独立修改并提交 PR。                                               |
| **Branch（分支）**                | 并行开发线，每个人在自己的分支上工作，完成后再合并到 `main`。                                               |
| **PR（Pull Request，拉取请求）**  | 请求将某分支的更改合并到目标分支，包含 Code Review 流程。                                                   |
| **Merge（合并）**                 | 将分支的更改合并到另一个分支。                                                                              |
| **Rebase（变基）**                | 将分支提交逐一应用到目标分支顶端，保持线性历史。                                                            |
| **Squash（压缩）**                | 将分支上所有提交压缩为一个提交再合并。                                                                      |
| **冲突（Conflict）**              | 两人修改同一文件同一区域时 Git 无法自动合并，需手动解决。冲突标记：`<<<<<<< HEAD` / `=======` / `>>>>>>>`。 |
| **合并编辑器（Merge Editor）**    | VS Code 提供的三栏编辑器（传入/合并结果/当前），用于解决复杂冲突。                                          |
| **.gitignore**                    | Git 忽略规则文件，指定不需要纳入版本控制的文件和目录。                                                      |

## Obsidian 相关

| 术语                | 说明                                                                                                       |
| :------------------ | :--------------------------------------------------------------------------------------------------------- |
| **Vault（仓库）**   | Obsidian 中的笔记仓库概念，即一个文件夹被 Obsidian 打开作为知识库。                                        |
| **.obsidian**       | Obsidian 配置目录，存放主题、插件、设置、工作区状态等。                                                    |
| **Canvas（画布）**  | Obsidian 内置可视化编辑工具，在无限画布上自由放置卡片、连接线、图片和网页嵌入。`.canvas` 文件本质是 JSON。 |
| **卡片（Card）**    | Canvas 中的基本单元，可包含文本、嵌入笔记、图片、网页等。                                                  |
| **连接线（Edge）**  | Canvas 中表示卡片间关系的箭头或直线，可调整颜色、线宽、线型。                                              |
| **分组（Group）**   | Canvas 中将多个卡片组合为逻辑区域，用半透明背景框围绕。                                                    |
| **Advanced Canvas** | 社区插件，增强 Canvas 能力：`Ctrl+F` 搜索、演示模式、Frontmatter 支持等。                                  |
| **Bases（数据库）** | Obsidian 内置结构化数据管理工具，以数据库形式组织和查询笔记内容。`.base` 文件本质是 YAML 配置。            |
| **Base**            | 一个 `.base` 文件，定义数据来源（筛选范围）、展示字段和视图。                                              |
| **字段（Field）**   | 对应笔记的 Frontmatter 属性或文件元数据（如文件名、创建时间、文件大小）。                                  |
| **视图（View）**    | Base 支持四种视图：**表格（Table）**、**卡片（Cards）**、**列表（List）**、**地图（Map）**。               |
| **筛选（Filter）**  | 定义 Base 的数据范围，可按文件夹、标签等条件组合。                                                         |
| **排序（Sort）**    | 对字段列进行多级排序。                                                                                     |
| **公式（Formula）** | Base 中的计算字段，可派生新值（如状态图标、已创建天数）。                                                  |
| **汇总（Summary）** | Base 表格底部对数值字段进行合计、平均值等统计计算。                                                        |

## Syncthing 相关

| 术语                             | 说明                                                                                                   |
| :------------------------------- | :----------------------------------------------------------------------------------------------------- |
| **Syncthing**                    | 开源、去中心化的文件同步工具，多设备间实时同步文件夹，无需第三方云服务器。                             |
| **设备 ID（Device ID）**         | 每台 Syncthing 设备的唯一标识符（52 位字符），基于密钥对生成，用于设备间身份验证和配对。               |
| **文件夹共享（Folder Sharing）** | Syncthing 同步的基本单位，可选择将本地文件夹与其他设备共享。支持仅发送、仅接收、双向同步三种类型。     |
| **中继（Relay）**                | 当两台设备无法直接建立连接时（如 NAT 网络），通过中继服务器转发数据（端到端加密）。                    |
| **发现（Discovery）**            | 帮助设备在互联网上找到彼此，不传输文件数据。局域网内可关闭全局发现。                                   |
| **SyncTrayzor**                  | Windows 版 Syncthing 的图形界面包装，含系统托盘功能。                                                  |
| **Syncthing-Fork**               | Android 平台社区维护的 Syncthing 版本（GitHub: researchxxl/syncthing-android）。                       |
| **.stignore**                    | Syncthing 的忽略规则文件（类似 `.gitignore`），排除不需要同步的文件。                                  |
| **.stversions**                  | 版本控制目录，存放旧版本文件副本。                                                                     |
| **冲突副本（.sync-conflict）**   | 当同一文件在两设备上同时修改时，Syncthing 生成冲突副本。格式：`原文件名.sync-conflict-时间戳-设备名`。 |
| **简单版本控制**                 | Syncthing 内置策略，保留最近 N 个被替换或删除的文件版本。                                              |

## VS Code / Copilot 相关

| 术语                              | 说明                                                                                                                 |
| :-------------------------------- | :------------------------------------------------------------------------------------------------------------------- |
| **Copilot**                       | GitHub 与 OpenAI 合作的 AI 编程助手，提供代码补全、对话交互、代码解释、重构建议等能力。                              |
| **Chat 面板**                     | 专门的对话窗口（`Ctrl+Alt+I`），支持追问、上下文引用、图片理解。                                                     |
| **内联对话（Inline Chat）**       | 按 `Ctrl+I` 在不离开编辑器的情况下与 AI 交互。                                                                       |
| **Slash Commands**                | Chat 中的 `/` 命令，如 `/help`、`/explain`、`/fix`、`/tests`、`/doc`。                                               |
| **指令文件（Instruction）**       | Markdown 文件，自动加载到对话上下文，让 AI 了解项目规范和编码风格。                                                  |
| **Prompt 文件**                   | `.prompt.md` 文件，通过 `/` 斜杠命令手动调用，适合一次性可复用任务模板。                                             |
| **Skill（技能）**                 | 最强大的自定义单元——包含说明、脚本和资源的文件夹（`.github/skills/<name>/SKILL.md`），支持多步骤工作流和渐进式加载。 |
| **MCP（Model Context Protocol）** | 由 Anthropic 提出、被 VS Code Copilot 采纳的开放协议，定义 AI 模型与外部工具/数据源之间的标准通信方式。              |
| **MCP 服务器**                    | 遵循 MCP 协议的外部工具/服务，可通过扩展市场或 `settings.json` 配置接入。                                            |
| **Custom Agent**                  | 特定角色的子代理，文件格式 `.github/agents/*.agent.md`，通过 `/` 或对话中提及触发。                                  |
| **Hooks**                         | Agent 生命周期事件触发的确定性脚本，文件格式 `.github/hooks/*.json`。                                                |
| **applyTo**                       | 文件级指令中的 glob 模式属性，只有匹配此模式的文件被操作时，指令才会自动加载。                                       |
| **渐进式加载**                    | Skill 的 Token 优化策略：第一阶段发现（~100 tokens），第二阶段指令（<5000 tokens），第三阶段资源（不定）。           |
| **Tavily Search MCP**             | 专为 AI 优化的搜索 MCP 服务器，返回结构化结果（摘要+来源），支持中文搜索。                                           |
| **Awesome Copilot**               | GitHub 官方维护的 Copilot 资源精选列表，包含 Instructions、Prompts、Custom Agents、Skills 示例。                     |

## 项目协作相关

| 术语                            | 说明                                                                                                           |
| :------------------------------ | :------------------------------------------------------------------------------------------------------------- |
| **分支策略（Branch Strategy）** | `main` 分支稳定发布，功能分支命名 `feat/<描述>` 或 `docs/<section>`。                                          |
| **提交信息格式**                | `<type>(<scope>): <简短描述>`，推荐类型：`docs`、`feat`、`fix`、`style`、`refine`。                            |
| **PR 流程**                     | 从 `main` 创建分支 → 编写文档 → 推送 → 创建 PR → Code Review → 合并到 `main`。                                 |
| **编写规范**                    | UTF-8 编码 Markdown，统一 Frontmatter，每节包含：学习目标 → 前置条件 → 步骤 → 常见问题 → 练习任务 → 验收清单。 |
